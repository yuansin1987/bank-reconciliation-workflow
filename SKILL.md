# 银行流水发票对账工作流 (ClawFlow)

## 概述

基于 ClawFlow 的营业收入银行流水与发票自动对账工作流。将银行对账单（PDF）和发票导出数据（Excel）进行交叉比对，通过多步骤后台任务完成数据处理，最终输出标准化的 4 Sheet 对账 Excel 报告并推送至飞书。

## 工作流类型

ClawFlow 标准工作流（`workflows/银行流水发票对账_clawflow/`）

## 适用场景

- 月度营业收入对账
- 发票与银行收款核对
- 银行手续费汇总
- 未开票收入筛查

## 输入文件

| 文件 | 说明 |
|---|---|
| 银行对账单 PDF | 招商银行对账单，包含期间所有收付款明细 |
| 全量发票查询导出结果.xlsx | 销项发票数据（Sheet2：发票信息） |

## 输入文件要求

- 发票 Excel 须包含 Sheet2（销项发票），列 D/I/J/K/L 分别为：发票号码/开票日期/金额/税额/价税合计
- 银行 PDF 须提取：日期、业务类型、摘要、借贷金额、对手户名
- 发票金额与银行收款金额以"价税合计（L列）"为匹配基准

## 输出文件结构

### Sheet1 — 发票收款对比（按公司）

| 列名 | 说明 |
|---|---|
| 序号 | 顺序编号 |
| 公司名称 | 银行对手方公司名称 |
| 当月收款金额 | 该公司当月银行收款总额 |
| 收款笔数 | 收款次数 |
| 收款日期 | 收款日期（逗号分隔） |
| 匹配发票金额 | 如与发票金额精确匹配则显示 |
| 发票张数 | 匹配发票数量 |
| 备注 | 已匹配 / 未匹配到发票 |

**匹配逻辑**：以"价税合计金额"精确匹配（金额相等到分）

**汇总**：发票张数 / 价税合计金额 | 银行收款总额 | 已匹配家数 | 未匹配家数

### Sheet2 — 银行手续费明细

包含：网银费用（跨行异地/本地/服务费）、对公转账服务费等

### Sheet3 — 收款未开票

当月银行已收款，但当月无对应发票金额的明细

### Sheet4 — 其他支出明细

包含：退款、工资、税款、差旅报销、其他服务费等

## 工作流步骤 (ClawFlow)

### Step 1 — 解析发票 Excel

读取 `全量发票查询导出结果.xlsx` → Sheet2（发票信息）

提取字段：D=发票号码, I=开票日期, J=金额(不含税), K=税额, L=价税合计

金额字段须用 `float()` 解析，排除空值

### Step 2 — 解析银行对账单 PDF

从 PDF 文本提取所有交易记录，整理为列表：

```
(date, type, desc, amt, counter)
```

- `amt > 0` 表示收款（贷方）
- `amt < 0` 表示付款（借方）
- 手续费识别：业务类型含"网银费用"或摘要含"手续费"

### Step 3 — 数据分组汇总

- **发票按金额汇总**：以 `L列（价税合计）` 为键，计算每个金额的发票价税合计合计
- **银行按公司汇总**：以 `counter（对手户名）` 为键，汇总每家公司收款总额

### Step 4 — 金额匹配

- 构建发票金额集合 `inv_pos`
- 对每笔银行收款金额精确匹配（`round(amt, 2) in inv_pos`）
- 已匹配：收款金额 = 发票价税合计
- 未匹配：收款金额无对应发票

### Step 5 — 生成 Excel

使用 Python zipfile 手动构建 XLSX（无第三方依赖）

表头行样式：蓝底白字粗体，数据行：带边框，金额列：数字格式

### Step 6 — 推送飞书

调用飞书 API 上传文件并发送给用户（open_id: `ou_57883f64459bea7339fc6c78d094d0e1`）

## 核心 Python 脚本

| 文件 | 作用 |
|---|---|
| `read_invoice.py` | 发票 Excel 解析脚本 |
| `gen_bank_xl.py` | 生成银行 Excel 模板/核心生成脚本 |
| `send_xl.py` | 飞书上传发送脚本 |

### read_invoice.py

读取发票 Excel，输出发票金额列表（价税合计 L 列）。

### gen_bank_xl.py

依赖：Python 3 标准库（zipfile, io, collections）

输入：`bank_data`（银行交易列表）, `inv_amts`（发票金额列表）

输出：4 Sheet Excel 文件

主要函数 `make_xlsx(out_path, sheets)`:
- `xl_esc(s)` — XML 转义
- `col_letter(n)` — 列号转字母（A=1→A, B=2→B...）
- `make_xlsx(out_path, sheets)` — 生成 XLSX 文件

### send_xl.py

使用飞书开放 API 上传 Excel 文件并发送给指定用户。

## 关键参数

| 参数 | 说明 |
|---|---|
| 发票张数 | Sheet2 发票行数 |
| 发票价税合计 | L 列金额之和 |
| 银行收款总额 | 所有 amt>0 且非利息的交易之和 |
| 银行手续费 | 各类网银费用 + 对公转账服务费之和 |
| 匹配家数 | 收款金额精确匹配发票的家数 |
| 未匹配家数 | 收款金额无对应发票的家数 |

## 下月复用方法

1. 替换 `gen_bank_xl.py` 中的 `bank_data` 列表（当月银行交易数据）
2. 替换 `inv_amts` 列表（当月发票价税合计金额）
3. 替换 `inv_count` 和 `inv_total_sum`（当月汇总数据）
4. 修改 `out_path` 桌面路径和月份
5. 运行 `py -3 gen_bank_xl.py`
6. 用 `send_xl.py` 上传至飞书
